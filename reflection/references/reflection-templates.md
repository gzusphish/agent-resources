# Reflection Templates

Templates for common reflection scenarios.

## Post-Task Reflection

Use after completing any significant task.

```markdown
## Reflection: [task-name]
**Date**: [YYYY-MM-DD]
**Duration**: [time taken]
**Steps**: [number of tool calls]
**Outcome**: [success/partial/failure]

### What Worked
- [Effective approach]
- [Tool that delivered]
- [Pattern to replicate]

### What Didn't
- [Inefficiency]
- [Wrong assumption]
- [Missed shortcut]

### Decision Analysis
1. [Decision point]
   - Choice: [what was done]
   - Outcome: [result]
   - Verdict: [good/ok/poor]

### Tool Efficiency
- Redundant actions: [count and describe]
- Could have used: [better approach]
- Optimal sequence: [ideal path for next time]

### User Alignment
- Explicit feedback: [what user said]
- Implicit signals: [tone, corrections]
- Improvement: [better approach next time]

### Action Items
- [Specific change to implement]
- [Checklist to add]
- [Skill to update]
```

## Error Recovery Reflection

Use after significant errors or failures.

```markdown
## Reflection: Error Recovery - [task-type]
**Date**: [YYYY-MM-DD]
**Error Type**: [tool/assumption/logic/context]

### Error Chain
1. Initial trigger: [what started it]
2. Escalation: [how it compounded]
3. Resolution: [how it was fixed]

### Root Cause Analysis
- Surface cause: [what appeared to happen]
- Deep cause: [underlying reason]
- Systemic issue: [if this is a pattern]

### Recovery Assessment
- Detection speed: [how quickly caught]
- Recovery time: [effort to fix]
- User impact: [disruption level]

### Prevention
- Early warning: [sign to watch for]
- Verification step: [check before proceeding]
- Alternative path: [better approach]
```

## Tool Choice Reflection

Use when tool selection was questionable.

```markdown
## Reflection: Tool Selection
**Task**: [description]
**Tools Used**: [sequence]
**Better Alternative**: [what should have been used]

### Analysis
- Why this choice: [reasoning at the time]
- Why it was suboptimal: [downsides]
- Decision heuristic: [rule for next time]

### Patterns
- Overuse: [tools used too often]
- Underuse: [tools not utilized enough]
- Blind spots: [capabilities missed]
```

## User Interaction Reflection

Use after notable user interactions.

```markdown
## Reflection: User Interaction
**Context**: [task type]
**User Signals**: [explicit and implicit]

### Communication Assessment
- Clarity: [did user understand suggestions?]
- Appropriateness: [right level of detail?]
- Timing: [when were check-ins needed?]

### Style Alignment
- Formality: [match with user?]
- Verbosity: [too much/too little?]
- Proactivity: [right balance?]

### Adjustments
- For this user: [specific preferences noted]
- For this task type: [approach refinements]
```
