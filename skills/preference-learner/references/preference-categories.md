# Preference Categories

Standard taxonomy for organizing user preferences.

## Code Style

### Formatting
- Indentation (tabs/spaces, count)
- Line length limits
- Quote style (single/double)
- Trailing commas preference
- Semicolon usage

### Structure
- Function size limits
- File organization patterns
- Import ordering
- Comment style/density

### Patterns
- Preferred abstractions
- Design pattern preferences
- Error handling approach
- Null/undefined handling

## Communication Style

### Verbosity
- Preferred level of detail
- Summary vs. comprehensive
- Code vs. explanation balance

### Formality
- Professional vs. casual tone
- Technical depth level
- Use of examples

### Interaction Frequency
- Check-in preferences
- Progress update timing
- Silent vs. verbose mode

## Tool Usage

### Preferred Tools
- Search tools priority order
- Editing approach (manual/bulk)
- Validation preferences

### Tool Avoidance
- Tools user dislikes
- Approaches to avoid
- Alternative preferences

### Automation Level
- Comfort with automated changes
- Review checkpoints needed
- Trust thresholds

## Workflow Preferences

### Order of Operations
- Plan first vs. dive in
- Test early vs. test late
- Documentation timing

### Checkpoints
- When to pause for review
- What needs confirmation
- Scope change handling

### Scope Handling
- Stay strictly in scope vs. identify related issues
- How to handle discovered problems
- Refactoring tolerance

## Output Format

### File Structure
- Directory organization preferences
- Naming conventions
- Where to put new files

### Documentation Style
- README preferences
- Code comment expectations
- External doc needs

### Deliverables
- Code vs. explanation balance
- Example requirements
- Testing expectations

## Detection Guide

### Explicit Signals
- "I prefer...", "I like...", "Always...", "Never..."
- Direct instructions about approach
- Style guide references

### Implicit Signals
- Repeated rejections of suggestions
- Consistent corrections in one direction
- Requests for alternatives
- Tone changes (frustration, enthusiasm)

### Pattern Signals
- Same correction multiple times
- Consistent approval of certain approaches
- Referencing past successful work
